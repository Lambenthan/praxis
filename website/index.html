For social-science researchers

From a research question to submission-ready work.

Fishes is a local-first desktop app that runs your quantitative, qualitative, and mixed-methods work — regressions you adjudicate into a final table, publication-grade figures, coding you rule on, manuscripts compiled to journal PDF and Word. No terminal. No coding agent to configure. Your data stays on your machine.

What it does

Three lanes — and you make every call.

The agent proposes; you decide. Every result lands as a file you own, and traces back to the code that made it.

Quantitative

Regressions you adjudicate

Hand it a dataset. It runs the data check, a baseline model menu and robustness, and hands you the models as candidates — you adopt the ones that make the final table. Every number traces to a runnable do-file; figures come out publication-grade by default.

Qualitative

Coding you rule on

Open-code a transcript into candidate codes you accept or reject one by one. Export to REFI-QDA (.qdpx) for NVivo and MAXQDA — the human stays in charge of the interpretation.

Writing

Journal PDF and Word

Compile a review or paper to a journal PDF (LaTeX) and a journal Word document — Chinese social-science formatting or APA — from the same manuscript.

Your data

You can read exactly how your data is handled.

The app's shell is open source for one reason: trust should be verifiable, not promised. Every guarantee below points to the code that implements it.

Your files and raw data stay on your machine. The agent runs inside the folder you chose, never your filesystem root.
runtime.rs · spawn_sidecar
Provider keys are stored locally, in an owner-only file. Never in the workspace, provenance, or exports.
runtime.rs · tighten_private
Keys are redacted from the debug log, and the log is owner-only — a bug report can't leak a credential.
debug_log.rs · redact
No telemetry, no analytics, no phone-home. Data leaves only during a conversation turn — nothing in the background.
grep: no posthog / sentry
The runtime is localhost-only and password-gated. A fresh random password each launch, held in memory, never on disk.
runtime.rs · server_password
Dangerous actions ask first. Deleting files, installs, remote connections, and web fetches prompt for approval by default.
opencode_config.rs · DANGEROUS_BASH

The only thing sent off-machine is your request to the model provider you chose — the same data you'd send using that provider's own website. The app states this in plain language under Settings → Privacy & data flow.

Open-core

The trust layer is open. The craft is ours.

Fishes is open-core — a deliberate split between the part you must be able to audit and the methodology that makes it good.

Open — the shell (MIT)

The desktop app, the interface, and the runtime integration layer: how keys are stored, how the agent is sandboxed to your workspace, and exactly what leaves your machine. This is what you need to trust, so it is open and auditable.

Proprietary — the methodology

The skills that do the annotation, the qualitative coding, the regression adjudication, the journal formatting, and the methodology reviews. Shipped inside the app, refined over many iterations — that craft is the product.
